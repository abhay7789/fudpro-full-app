## Overview

FudPro (Mumbai Dabbawala System) is a modular monolith application designed to manage a food delivery ecosystem involving Customers, Vendors, and Admins. The system is split into a React-based frontend and a Node.js/Express-based backend, communicating via a RESTful API.

```
┌─────────────────────────────────────────┐
│              [React SPA]                │
│         (Vite + Mantine + Zustand)      │
├─────────────────────────────────────────┤
│              [REST API]                 │
│         (Express.js + Sequelize)        │
├─────────────────────────────────────────┤
│            [PostgreSQL DB]              │
└─────────────────────────────────────────┘
```

## Components

### Frontend (frontend/)
- **Core:** React 19 with Vite 8.
- **UI Framework:** Mantine 9 for components and styling.
- **State Management:** Zustand for global application state.
- **Animations:** Framer Motion for premium UI interactions.
- **Routing:** React Router Dom 7 for client-side navigation.
- **I18n:** react-i18next for multi-language support.

### Backend (backend/)
- **Core:** Express.js 4 with a modular monolithic architecture.
- **ORM:** Sequelize 6 interacting with PostgreSQL.
- **Modules:**
    - `auth`: User authentication and JWT management.
    - `user`: User profile and customer-specific logic.
    - `vendor`: Vendor onboarding and management.
    - `menu`: Menu items and category management.
    - `order`: Order lifecycle and fulfillment.
    - `payment`: Payment processing logic.
    - `delivery`: Delivery logistics and assignment.
    - `notification`: In-app and email notifications.
    - `admin`: Superadmin and admin dashboard logic.
    - `media`: Centralized file/media handling (Multer).
- **Security:** JWT (jsonwebtoken), bcryptjs, and role-based access control (RBAC).
- **Logging:** Winston for system logs and audit trails.
- **Documentation:** Swagger (swagger-ui-express) for interactive API documentation at `/api-docs`.

### Data Model
- **Schema:** Uses a dedicated `fudpro` schema in PostgreSQL.
- **Key Entities:** Users, Roles, Vendors, Categories, MenuItems, Orders, Payments, Deliveries.

## Data Flow

1. **User Interaction:** User interacts with the Mantine-based UI.
2. **Action Trigger:** UI components trigger logic which may call Zustand store actions or hooks.
3. **API Request:** Frontend sends HTTP requests using Axios to the `/api/v1` endpoints.
4. **Middleware:** Backend processes requests through loaders (Express, Routes) and shared middleware (Auth, Role, Error).
5. **Controller/Service:** Routes map to controllers which delegate business logic to services.
6. **Persistence:** Services use Sequelize models to query or mutate the PostgreSQL database within the `fudpro` schema.
7. **Response:** Backend returns JSON responses which the frontend uses to update its state and UI.

## Conventions

**Naming:** CamelCase for frontend components, camelCase for functions/variables, snake_case for DB fields (Sequelize handles underscored: true).
**Structure:** Module-based organization in `backend/src/modules` and `frontend/src/modules`.
**State:** Single source of truth for auth and session state in Zustand.
